# Semantic News Novelty

Semantic News Novelty — анализатор информационного потока для аналитических
агентств. Он работает с публикациями из источников клиента, группирует их в
инфоповоды, выявляет повторы и значимые обновления и предоставляет
семантический поиск для ускорения подготовки отраслевых отчётов.

`Semantic News Novelty` — публичное название продукта. `news-flow-analysis` —
технический идентификатор репозитория.

## Что делает система

Основная целевая аудитория — небольшие и средние аналитические агентства,
которые готовят отраслевые обзоры, тематические подборки и другие B2B-отчёты.
Система устанавливается поверх источников и данных клиента.

До содержательного анализа значительное время уходит на подготовительную
работу: чтение потока, сведение публикаций из разных источников, удаление
повторов и поиск изменений в уже знакомых сюжетах. Keyword search и таблицы
помогают организовать данные, но плохо отражают смысловую близость материалов.

Semantic News Novelty превращает собранный массив публикаций в рабочую
структуру, с которой аналитик может быстрее перейти к интерпретации материала.

Основной сценарий:

```text
Источники клиента
        ↓
Импорт публикаций
        ↓
Кластеризация и дедупликация
        ↓
Оценка значимых обновлений
        ↓
Семантический поиск и ручная проверка
        ↓
Подготовка отчёта аналитиком
```

## Возможности

Система:

- импортирует публикации из файлов и через HTTP API;
- объединяет материалы об одном инфоповоде;
- выявляет дубли и близкие пересказы;
- отмечает значимые обновления внутри сюжета;
- поддерживает семантический поиск с сокращением повторов в выдаче;
- позволяет вручную проверить и скорректировать результаты;
- обрабатывает первичную загрузку в режиме `full` и новые поступления в режиме
  `incremental`;
- предоставляет демонстрационный UI и API для интеграции.

Результатом является не готовый аналитический отчёт, а структурированная основа
для работы эксперта: кластеры инфоповодов, метки новизны, релевантные публикации
и история развития сюжета.

## Границы продукта

Semantic News Novelty работает с уже собранным информационным потоком клиента.
Проект не является:

- системой сбора и мониторинга всех внешних источников;
- универсальной системой мониторинга и анализа информационного поля;
- инструментом автоматического фактчекинга;
- генератором готовых отчётов без участия аналитика;
- заменой эксперта предметной области.

Качество необходимо проверять на источниках и тематике конкретного клиента.
Текущий технический результат основан на исследовательском датасете Lenta.ru и
не является гарантией переноса на новый домен.

## Архитектура

Docker Compose поднимает:

- `nginx` — единая точка входа;
- `ui` — Streamlit-интерфейс;
- `api` — FastAPI для пользователей, публикаций и операций;
- `model-service` — BGE-M3, кластеризация и оценка новизны;
- `rabbitmq` — очередь асинхронных задач;
- `postgres` — PostgreSQL 16 с pgvector.

Публикации хранятся в PostgreSQL. API передаёт через RabbitMQ только ID и режим
обработки, а `model-service` читает данные, выполняет pipeline и сохраняет
эмбеддинги, кластеры и оценки обратно в базу.

Runtime использует PostgreSQL/pgvector, а не FAISS.

## Быстрый запуск

Требуются Docker Compose и NVIDIA GPU для рекомендуемого режима.

Скопируйте `.env.example` в `.env`.

В `.env` замените пароли и секреты, затем включите демонстрационный режим:

```text
DEMO_MODE=true
MODEL_SERVICE_GPU_REPLICAS=1
MODEL_SERVICE_CPU_REPLICAS=0
```

Запуск:

```console
docker compose up --build -d
docker compose ps
```

После готовности:

- UI: <http://localhost/>
- OpenAPI: <http://localhost/api/docs>
- API health: <http://localhost/api/health>
- RabbitMQ: <http://localhost:15672/>

Первый запуск может занять несколько минут из-за загрузки BGE-M3. CPU-вариант, подключение кеша модели и эксплуатационные настройки описаны в [документе развёртывания](docs/deployment.md).

## ML-пайплайн

Актуальная конфигурация:

- векторизация: `BAAI/bge-m3`;
- строгая графовая кластеризация с временным окном;
- консервативный второй проход присоединения к лучшему кластеру;
- CatBoost-классификатор новизны;
- разметка `significant`, `minor`, `duplicate`;
- работа в пакетном `full` и последовательном `incremental` режимах.

Первая публикация нового кластера получает `significant` по детерминированному правилу. Неоднозначные назначения помечаются для ручной проверки.

## Ключевые результаты

На текущем небольшом golden-наборе:

| Метрика | Результат |
|---|---:|
| Pairwise F1 кластеризации | **0.9017** |
| False merge rate | **1.89%** |
| F1 значимых обновлений | **0.9114** |
| MRR@10 семантического поиска | **0.9944** |
| Снижение повторов в Top-10 | **75.14%** |

Прогретый GPU `full` на 10 000 публикаций: 226.33 с, или 44.18 публикации/с. Эти результаты не являются production-гарантией: golden ограничен 121 публикацией, а продуктовый эффект ещё должен быть проверен на данных аналитического агентства.

Подробности: [оценка качества](docs/evaluation.md) и
[бенчмарки](docs/benchmarks.md).

## Текущее состояние

Реализованы:

- импорт и жизненный цикл публикаций;
- асинхронный full/incremental pipeline;
- хранение BGE-M3 embeddings в pgvector;
- семантический поиск с дедупликацией по кластерам;
- роли, организации и учёт операций;
- Streamlit UI;
- GPU- и CPU-воркеры;
- тесты и CI.

Следующий этап — клиентский пилот на 300–500 публикациях:

- проверить перенос качества на источники клиента;
- оценить реальные поисковые запросы аналитиков;
- измерить сокращение времени подготовки отчёта;
- определить допустимый интервал полного reclustering.
