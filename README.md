# News Event Detection

## 1. Бизнес-анализ (Business Understanding)

### 📌 Цель проекта

Проект направлен на анализ новостного потока и автоматическое выделение событий.

В современных новостях наблюдается проблема:
- большое количество публикаций
- дублирование информации
- постепенное развитие событий через множество новостей

Цель проекта — разработать систему, которая позволяет:
- выделять ключевые события
- группировать связанные новости
- отслеживать развитие событий во времени

Бизнес-цель проекта — сократить время ручного анализа новостного потока за счёт автоматического объединения связанных публикаций в события, отделения дублей от обновлений и выделения новых инфоповодов.

---

### 👥 Организационная структура

**Заказчик (гипотетический):**
- новостной агрегатор / аналитическая платформа

**Пользователи:**
- аналитики
- журналисты
- PR-специалисты
- исследователи

Проект выполняется в формате индивидуальной разработки.

Роль заказчика моделируется как новостной агрегатор или аналитическая платформа.

---

### 🔍 Существующие решения

Существуют:
- новостные агрегаторы
- системы поиска новостей
- системы тематической классификации

Недостатки:
- не выделяют события, а только темы
- не учитывают развитие событий
- не различают дубликаты и обновления

### 📊 Данные

Используется:
- датасет парафразов для обучения модели
Дополнительно используется датасет русскоязычных новостных парафразов. Он нужен для дообучения embedding-модели под задачу поиска похожих публикаций.
- датасет новостей Lenta.ru (~800k новостей)
Используется как источник новостей

Данные доступны локально и не требуют внешнего доступа.

- автоматически (метрики)
- вручную (анализ результатов)

### ⚠️ Риски

1. **Не уложиться в сроки**
   - решение: упрощение модели и фокус на MVP

2. **Низкое качество модели**
   - решение: использование предобученных моделей

3. **Отсутствие явной структуры событий**

   - решение: ограничение по времени

## 1.2 Data Mining Goals

### 🎯 Формализация задачи

Задача:

> Определить, относится ли новость к существующему событию, парафразом или является новой.

---

### ⚙️ Подзадачи

1. Преобразование текста в embeddings  
2. Поиск похожих новостей  
3. Группировка новостей в события  
4. Определение роли новости:
   - новое событие
   - обновление
   - дубликат  

---
### Обучение модели

Для адаптации модели к задаче поиска похожих новостей используется датасет парафразов.

Идея обучения:

- anchor — исходный новостной заголовок
- positive — его парафраз / близкая новость
- negative — нерелевантная новость

Модель дообучается так, чтобы похожие новости, их парафразы располагались ближе друг к другу в embedding-пространстве, а непохожие — дальше.
---

### 📏 Метрики

Используются:

- Recall@K (поиск похожих новостей)
- MRR (качество ранжирования)

Дополнительно:
- ручная оценка качества кластеров

---

### 🎯 Критерий успешности

- Recall@1 > 0.9 на задаче поиска дублей  
- формирование осмысленных событийных кластеров  
- корректное разделение событий по времени  

---

## 1.3 План проекта (Project Plan)

План проекта соотнесён с основными фазами внедрения аналитического решения: бизнес-анализ, анализ данных, подготовка данных, моделирование, оценка результатов и финализация MVP.

### Этап 1. Бизнес-анализ
- формализация бизнес-цели
- определение пользователей и сценариев использования
- описание рисков и ограничений проекта

---

### Этап 2. Анализ и подготовка данных
- подготовка датасета русскоязычных новостных парафразов для дообучения embedding-модели
- загрузка и очистка Lenta dataset
- фильтрация и нормализация
- анализ структуры данных, временного диапазона, тем и пропусков

---

### Этап 3. Построение embeddings и моделирование
- использование предобученной transformer-модели для получения семантических векторов новостей
- дообучение модели на подготовленных парах новостных парафразов
- обучение выполняется в формате metric learning: похожие новости/парафразы должны располагаться ближе в векторном пространстве
- в качестве функции потерь используется Multiple Negatives Ranking Loss / Triplet Loss (как потенциальное расширение)

---

### Этап 4. Поиск похожих новостей и группировка
- построение FAISS индекса
- реализация nearest neighbor поиска
- разработка алгоритма объединения в события
- добавление временного ограничения

---

### Этап 5. Оценка результатов
- визуальная проверка кластеров
- оценка качества поиска похожих новостей через Recall@K и MRR
- ручная проверка осмысленности событийных кластеров

---

### Этап 6. Финализация и MVP
- подготовка отчёта
- оформление результатов
- подготовка воспроизводимой структуры проекта и базового сервиса

---
