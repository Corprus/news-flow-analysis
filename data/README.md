# Данные

Полные датасеты не хранятся в Git-репозитории.

В репозитории должны находиться только код, документация, конфигурация и небольшие тестовые примеры. Исходные и подготовленные данные могут быть большими, поэтому они хранятся локально и исключены из Git через `.gitignore`.

## Ожидаемая структура

```text
data/
  raw/
    lenta.csv
  prepared/
```

## Lenta.ru dataset

Основной датасет новостей ожидается по пути:

```text
data/raw/lenta.csv
```

Ожидаемые колонки:

- `url`
- `title`
- `text`
- `topic`
- `tags`
- `date`

Датасет используется как источник новостного потока для построения embeddings, поиска похожих новостей и группировки публикаций в события.

## Датасет парафразов

Для дообучения embedding-модели используется датасет:

```text
merionum/ru_paraphraser
```

Локальная копия может сохраняться в:

```text
data/raw/ru_paraphraser_hf
```

Подготовленные пары парафразов могут сохраняться в:

```text
data/prepared/
```

Для воспроизводимого дообучения модели используется скрипт:

```text
scripts/train_embeddings.py
```

## Тестовые данные

Небольшие синтетические или обезличенные примеры для тестов можно хранить в:

```text
tests/fixtures/
```

Такие файлы должны быть маленькими и не должны содержать полный исходный датасет.
