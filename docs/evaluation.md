# Оценка качества

Этот документ является каноническим источником метрик качества модели.
Производительность описана отдельно в [бенчмарках](benchmarks.md).

## Данные

Исследовательский контур построен на новостях Lenta.ru. Используются:

- golden-набор с ручной экспертной разметкой;
- silver-набор с LLM-разметкой для обучения и weak-positive настройки;
- bronze-выгрузки для расширения ручной разметки.

Параметры `exp_10` выбирались без golden-набора. Golden использовался как
отложенная проверка после фиксации конфигурации.

Размер текущей проверки ограничен:

- 121 публикация для кластеризации;
- 87 строк с доступной разметкой новизны;
- 119 синтетических поисковых запросов из заголовков golden-публикаций.

Поэтому результаты подтверждают внутренний технический критерий, но не
гарантируют качество на домене клиента.

## Кластеризация

| Метрика | Baseline `exp_00b` | Финальный `exp_10` |
|---|---:|---:|
| Pairwise precision | 1.0000 | 0.9811 |
| Pairwise recall | 0.6257 | 0.8342 |
| Pairwise F1 | 0.7697 | **0.9017** |
| False merge rate | 0.00% | **1.89%** |

Второй проход заметно уменьшает фрагментацию сюжетов ценой трёх ошибочно
объединённых пар из 159 предсказанных same-story пар.

## Значимые обновления

Метрики включают runtime-правило «первая публикация кластера →
`significant`».

| Метрика | Результат |
|---|---:|
| Precision significant | 0.8471 |
| Recall significant | 0.9863 |
| F1 significant | **0.9114** |

Эти значения совпадают для baseline- и финальной кластеризации на текущем
размеченном подмножестве. Измеренное улучшение `exp_10` относится к структуре
кластеров, а не к novelty classifier.

## Семантический поиск и дедупликация

| Вариант | MRR@10 | Повторы в Top-10 | Снижение повторов |
|---|---:|---:|---:|
| TF-IDF keyword baseline | 0.9067 | 46.64% | — |
| BGE-M3 без cluster collapse | 0.9926 | 51.51% | −10.45% |
| BGE-M3 + `exp_10` cluster collapse | **0.9944** | **11.60%** | **75.14%** |

Семантическое ранжирование улучшает MRR, но само по себе не устраняет повторы.
Основной эффект дедупликации даёт collapse результатов по кластерам.

Поисковая проверка является smoke-test: запросы синтетические и построены из
заголовков того же небольшого корпуса. Для продуктового вывода нужны реальные
запросы аналитиков и независимая поисковая разметка.

## Внутренние критерии MVP

| Требование | Порог | Получено |
|---|---:|---:|
| Pairwise F1 кластеризации | ≥ 0.75–0.80 | **0.9017** |
| False merge rate | ≤ 10–15% | **1.89%** |
| F1 значимых обновлений | ≥ 0.75 | **0.9114** |
| Recall значимых обновлений | ≥ 0.80 | **0.9863** |
| MRR@10 | выше keyword baseline | **0.9944 против 0.9067** |
| Снижение повторов Top-10 | ≥ 30% | **75.14%** |

Технические критерии на текущем наборе достигнуты.

## Что ещё не подтверждено

- перенос на новые тематики, источники и периоды;
- стабильность на существенно большем независимом корпусе;
- полезность меток для конкретного аналитического процесса;
- качество на реальных поисковых запросах;
- сокращение времени подготовки отчёта.

Следующая обязательная проверка — пилот на 300–500 публикациях клиента с
экспертной оценкой.
