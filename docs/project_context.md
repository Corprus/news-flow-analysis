# Контекст проекта

## Продуктовая формулировка

Semantic News Novelty — модуль постобработки уже собранного открытого информационного
потока для отраслевого desk research.

Основной пользователь — аналитик малого или среднего исследовательского агентства,
который готовит платные B2B-обзоры рынков и тематические подборки. Система не заменяет
аналитика и не занимается full-cycle мониторингом источников. Она формирует черновую
рабочую структуру:

- группирует публикации в инфоповоды;
- находит дубли и близкие пересказы;
- выделяет значимые обновления;
- поддерживает семантический поиск по базе клиента.

## Ценность

Текущий процесс основан на ручной вычитке, таблицах, RSS, Telegram и keyword search.
Проект должен сократить рутинную часть работы и позволить эксперту быстрее перейти к
интерпретации материала и подготовке обзора.

Время ручной вычитки рассматривается как ожидаемый эффект пилота, а не как
воспроизводимый ML-benchmark: результат зависит от аналитика, темы и входного потока.

## Граница MVP

В MVP входят:

- импорт уже собранных публикаций;
- событийная кластеризация;
- дедупликация;
- оценка смысловой новизны внутри сюжета;
- семантический поиск;
- выгрузка результата через CSV, API или интерфейс.

В MVP не входят:

- самостоятельный сбор всего открытого интернета;
- замена полноценной media intelligence платформы;
- автоматический фактчекинг;
- обязательная LLM-суммаризация и генерация готового отчёта.

## Данные и оценка

Исследовательский контур построен на новостях Lenta.ru:

```text
data/raw/lenta.csv
data/prepared/lenta_clean_news.csv
```

Для оценки используются:

- golden-набор с ручной экспертной разметкой;
- silver-набор с LLM-разметкой для обучения и weak-positive настройки;
- bronze-выгрузки для расширения разметки.

Golden не использовался для выбора параметров `exp_10`, чтобы сохранить его holdout
оценкой.

## Текущий ML pipeline

Финальный исследовательский pipeline:

1. `BAAI/bge-m3` embeddings;
2. строгая графовая кластеризация с threshold `0.82` и окном 14 дней;
3. консервативный `best-candidate attach`;
4. previous-only novelty features без утечки из будущего;
5. CatBoost classifier для элементов с предыдущим контекстом в кластере;
6. детерминированное правило `cluster seed → significant` и постобработка
   `minor`, `duplicate`, `needs_review`.

На golden-наборе финальная конфигурация получила:

- Pairwise F1 кластеризации: `0.9017`;
- false merge rate: `0.0189`;
- precision значимых обновлений: `0.8471`;
- recall значимых обновлений: `0.9863`;
- F1 значимых обновлений: `0.9114`.

Novelty-метрики включают runtime-правило `cluster seed → significant`. Улучшение
`exp_10` относительно baseline относится к кластеризации; novelty-метрики при одной
модели и одинаковой постобработке совпадают.

Размер golden-набора ограничен 121 публикацией, поэтому следующий обязательный этап —
пилот на 300–500 публикациях клиента и проверка переноса на новые темы и источники.

## Сервисный контур

Отдельный runtime-контур реализует:

- FastAPI API;
- PostgreSQL с pgvector;
- RabbitMQ jobs;
- model-service с `BAAI/bge-m3` и финальным clustering/novelty pipeline;
- Streamlit UI и nginx.

Он поддерживает векторизацию, хранение, кластеризацию, оценку новизны и семантический
поиск.

## Позиционирование и монетизация

Модель распространения — B2B SaaS с пилотом и возможной калибровкой порогов под клиента.
Цена может зависеть от объёма документов, количества тем и глубины истории. Решение
должно быть уже и дешевле full-cycle платформ уровня Feedly Market Intelligence,
NewsWhip или российских систем медиамониторинга.

Ключевой продуктовый риск: клиент может быть удовлетворён текущим процессом или не
считать выигрыш по качеству и времени достаточным для покупки отдельного сервиса.
